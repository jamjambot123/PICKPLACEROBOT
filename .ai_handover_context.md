# AI Handover Context & Project Checkpoint
**Last Updated:** 2026-06-10 (Simulation Phase Complete)

이 파일은 다른 AI 모델로 프로젝트를 인계하거나, 시간이 지난 후 기존 AI가 다시 돌아왔을 때 **프로젝트의 현재 상태와 맥락을 즉각적으로 파악하기 위한 체크포인트**입니다.

## 1. 현재 달성한 상태 (Current Status)
- **물리 엔진 (완료)**: Perfect Servo + Flexible Load 기반의 2-Mass Model 구현 (`input-shaping.js`). 잔류 진동이 0.06μm 수준으로 실제 산업 기준에 맞게 정밀 검증됨.
- **UI/UX 오버홀 (완료)**: 기존 네온/게이밍 스타일을 버리고 MATLAB/Simulink와 유사한 라이트/다크 그레이 전문 엔지니어링 테마로 전환 (`style.css`).
- **3D 시뮬레이터 (완료)**: 메인 로봇(Shaped)과 고스트 로봇(Unshaped)을 동시에 렌더링하여 시작부터 도착 후까지 진동의 차이를 직관적으로 비교 (`scene.js`). 모션 속도 조절(0.1x ~ 3x) 및 자동 슬로우 모션 탑재.
- **AI 옵티마이저 (완료)**: 목적 함수가 도착 이후의 잔류 진동만을 평가하도록 수정됨. 최대 UPH를 달성하는 파라미터 자동 튜닝 구현 (`ai-optimizer.js`).
- **3-프로파일 비교 모드 (완료)**: 
  - (Part A) 동일 조건에서의 Trapezoidal / AS-Curve / ZVD 비교.
  - (Part B) 각 기술별 AI 최대 성능 최적화 비교.
  - 결과를 표 형태로 보여주고 `.csv`로 다운로드하는 기능 구현 (`app.js`).

## 2. 절대 훼손하면 안 되는 핵심 원칙 (Core Principles)
1. **변인 통제와 물리적 정확도**: 이 시뮬레이터는 '증명'을 위한 도구입니다. 애니메이션의 화려함을 위해 물리 엔진의 수식이나 차트 결과값을 임의로 조작하거나 페이크(Fake) 연산을 넣어서는 절대 안 됩니다.
2. **비교의 직관성**: 제어 적용(Shaped)과 미적용(Unshaped)의 차이가 한 화면에서 명확히 비교되어야 합니다.
